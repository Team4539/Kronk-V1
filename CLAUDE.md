# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Kronk** — FRC Team 4539's 2026 competition robot. WPILib 2026.2.1 (Java 17), CTRE Phoenix 6 swerve drivetrain, PathPlanner autonomous, PhotonVision AprilTag pose estimation. Fixed shooter (no turret) — the entire robot rotates to aim.

## Build & Deploy Commands

```bash
./gradlew build          # Compile robot code
./gradlew deploy         # Deploy to roboRIO (requires network connection to robot)
./gradlew simulateJava   # Run simulation with Driver Station GUI
```

## Architecture

### Shooting Pipeline (all on-board, no coprocessor)

1. PhotonVision camera detects AprilTags → robot pose
2. `ShootingCalculator` (singleton) computes distance to target, heading offset, and flywheel RPM via linear interpolation from `Constants.Shooter.SHOOTING_CALIBRATION`
3. `RobotContainer.updateVisionPose()` calls `shootingCalculator.update()` every 20ms cycle
4. Commands read from `ShootingCalculator` — they never calculate independently

Auto-aim is integrated into the **default drive command**, not a separate command. When LT > 0.5, right-stick rotation is replaced by a PD controller (`aimOmega()` in RobotContainer) that rotates toward the target.

### Game State System

`GameStateManager` (singleton) tracks alliance color, alternating 25-second scoring shifts, and target mode (HUB vs TRENCH). All field coordinates have BLUE and RED variants, auto-selected based on alliance. PathPlanner paths are drawn for Blue origin; AutoBuilder flips for Red automatically.

### Subsystem Nullability

All subsystems in `RobotContainer` are nullable, guarded by null-checks, and conditionally initialized via `Constants.SubsystemEnabled` flags. This allows testing individual subsystems without full hardware.

### Stall Detection

Shooter, Trigger, and Intake all have stall detection state machines. If a motor can't deliver expected RPM for a timeout period, it auto-reverses to clear jams.

## Critical Gotchas

- **`generated/TunerConstants.java`** is generated by Phoenix Tuner X — never edit by hand
- **Single Xbox controller** (port 0) for ALL inputs — no separate operator controller
- **RPM calibration table** uses `List<double[]>` as `{distanceMeters, shooterRPM}`, linearly interpolated and extrapolated beyond table bounds
- **LED strip layout**: 8 onboard (indices 0-7) + 38 strip (8-45); belly strip is disconnected
- **Named commands** must be registered in `RobotContainer.registerNamedCommands()` before the auto chooser is built

## Key Files by Task

| Task | Files |
|------|-------|
| Tune shooting | `Constants.Shooter.SHOOTING_CALIBRATION`, `CalibrationManager`, `ShootingCalculator` |
| Fix auto-aim | `RobotContainer.aimOmega()`, `Constants.Driver.AIM_*` |
| Modify autos | `pathplanner/autos/*.auto`, `pathplanner/paths/*.path`, `RobotContainer.registerNamedCommands()` |
| Add a subsystem | New class in `subsystems/`, add to `Constants.CANIds`, `Constants.SubsystemEnabled`, `RobotContainer` |
| Change LEDs | `LEDSubsystem.java` (priority-based state machine) |
| Tune swerve | `generated/TunerConstants.java` (via Tuner X app), `pathplanner/settings.json` |

## Dashboard

Uses `DashboardHelper.Category` enum (`PRE_MATCH`, `AUTO`, `TELEOP`, `POST_MATCH`, `SETTINGS`, `DEBUG`) to organize SmartDashboard entries. `Elastic.sendNotification()` sends rich notifications to Elastic Dashboard.
